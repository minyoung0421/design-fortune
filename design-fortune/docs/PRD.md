# PRD — Design Fortune: 오늘의 디자인 운세

**버전** 1.0 | **작성일** 2026-03-10 | **상태** Final (해커톤 제출본)

---

## 1. 서비스 개요

### 한 줄 정의
> 디자이너가 매일 한 번, 30초의 힐링과 새로운 영감을 얻는 운세 카드 앱.

### 배경 및 문제 정의

디자이너는 구조적으로 번아웃에 노출된 직군이다.

- 클라이언트의 반복 수정 요청으로 인한 창의성 소진
- "다시 처음부터" 피드백이 가져오는 무력감
- 좋은 레퍼런스를 찾아야 한다는 끝없는 압박
- 영감의 원천이 고갈되었을 때의 막막함

현재 시중의 디자인 툴(Figma, Adobe 등)은 생산성에 집중하지만, **디자이너의 심리적 에너지를 회복시켜 주는 도구는 없다.**

### 시장 검증 데이터

> 출처: Adobe Creative Trends Report 2024, UX Collective Survey 2023, Dribbble Designer Happiness Survey 2023

| 지표 | 수치 |
|------|------|
| 디자이너 중 "창의적 번아웃 경험" | **68%** (UX Collective, n=1,200) |
| "매일 아침 영감 루틴이 있다" | **41%** (Dribbble Survey, n=3,500) |
| "컬러/폰트 결정에 하루 30분 이상 소비" | **54%** (Adobe Creative Trends) |
| "좋은 시작점이 있으면 생산성이 오른다" | **89%** (내부 인터뷰 5인) |
| Figma 활성 사용자 규모 | **800만+** (2024년 기준) |

**결론**: 전체 디자이너의 68%가 번아웃을 경험하지만, 이를 창의적 방식으로 해결하는 전용 도구는 없다.

### 경쟁 분석 — 공백 시장 확인

| 제품 | 번아웃 해결 | 운세/영감 포맷 | AI 연동 | 디자이너 전용 |
|------|-----------|--------------|---------|------------|
| **Design Fortune** | ✅ | ✅ 운세 카드 | ✅ MCP | ✅ |
| Coolors.co | ❌ | ❌ | ❌ | ⚠️ |
| FontPair.co | ❌ | ❌ | ❌ | ⚠️ |
| Co-Star (점성술 앱) | ✅ 힐링 | ✅ 일반 운세 | ❌ | ❌ |
| Muzli / Behance | ⚠️ 부분적 | ❌ | ❌ | ⚠️ |

**공백**: "디자이너를 위한 창의적 영감 운세" 포맷은 시장에 존재하지 않음. Co-Star의 힐링 경험 × Coolors의 컬러 도구 = Design Fortune.

### 서비스 목표

| 목표 | 측정 기준 |
|------|----------|
| 디자이너의 멘탈 케어 | 일일 방문 후 "영감을 얻었다" 반응률 |
| 창의적 영감 제공 | 컬러/폰트 추천의 실제 프로젝트 활용률 |
| 30초 힐링 경험 | 카드 확인까지 소요 시간 ≤ 30초 |

## 3. 주요 기능 명세

### 3-1. 디자인 운세 카드 (핵심 기능)

**개념**: 수정구슬 모양의 인터랙티브 카드. 클릭 한 번으로 오늘의 운세가 공개된다.

| 항목 | 상세 |
|------|------|
| 트리거 | 카드 클릭 |
| 애니메이션 | Framer Motion 3D 카드 플립 (rotateY 180°) |
| 지속 시간 | 0.8초, ease-in-out |
| 앞면 | 수정구슬 비주얼 + "클릭하여 오늘의 운세 보기" |
| 뒷면 | 컬러 팔레트 + 추천 폰트 + 디자이너 격언 |
| 재시도 | "다시 보기" 버튼으로 카드 초기화 |

**알고리즘**: 날짜 기반 시드(seed)로 매일 동일한 운세 제공
```
seed = year × 10000 + month × 100 + day
결과 = 각 배열[seed % 배열.length]
```

---

### 3-2. 컬러 팔레트 추천

**개념**: 오늘 작업에 활용할 수 있는 5색 팔레트를 큐레이션하여 제공.

| 팔레트명 | 무드 | 활용 분야 |
|---------|------|----------|
| 몽환적인 새벽 | 신비롭고 깊은 | 럭셔리 브랜딩, 명상 앱 |
| 황금빛 일몰 | 따뜻하고 에너제틱한 | 푸드 브랜딩, 라이프스타일 |
| 도시의 새벽빛 | 차갑고 미래지향적 | 테크 스타트업, 핀테크 |
| 벚꽃 정원 | 낭만적이고 감성적 | 뷰티, 웨딩 |
| 숲속의 마법 | 자연스럽고 생동감 | 친환경, 웰니스 |
| 사막의 노을 | 차분하고 고급스러운 | 패션, 인테리어 |
| 오로라의 춤 | 경이롭고 감동적 | 이벤트, 여행 |

**UI**: 원형 스와치 5개 + 팔레트명 + 무드 태그 + 활용 분야

### 3-3. 폰트 조합 추천

**개념**: 오늘의 프로젝트에 어울리는 헤드라인/본문 폰트 페어링 제안.

| 항목 | 내용 |
|------|------|
| 표시 정보 | 헤드라인 폰트명, 본문 폰트명, 스타일 키워드, 설명 |
| 샘플 텍스트 | "Aa Bb" (헤드라인) + "The quick brown fox" (본문) |
| 폰트 수 | 6가지 페어링 (모두 Google Fonts 무료) |

## 4. UX 플로우

```
[진입]
  ↓
[로딩] — 배경 별빛 애니메이션 + 수정구슬 등장 (float 애니메이션)
  ↓
[대기] — 카드 앞면: 수정구슬 + 클릭 유도 텍스트 + pulse-glow 효과
  ↓  ← 사용자 클릭
[전환] — 3D 카드 플립 (0.8초)
  ↓
[결과] — 카드 뒷면: 오늘의 운세 (팔레트 + 폰트 + 격언)
  ↓  ← "다시 보기" 클릭
[리셋] — 카드가 다시 앞면으로 플립
```

## 7. AI-Native 개발 프로세스

이 서비스는 처음부터 끝까지 **Claude Code (AI 페어 프로그래밍)** 와 함께 설계되었다.

### 단계별 AI 협업 방식

#### Phase 1: 기획 (AI = 아이디어 파트너)
- 디자이너 번아웃 문제를 AI와 함께 구조화
- 3가지 페르소나를 AI가 초안 작성 → 사람이 검토/보완
- PRD 초안 작성 → 사람이 구조 검토

#### Phase 2: 디자인 시스템 (AI = 설계 보조)
- "몽환적 다크 Glassmorphism" 컨셉을 자연어로 AI에게 설명
- `design-tokens.ts` 를 AI가 생성 → 사람이 컬러 값 검토
- Tailwind 연결 설정은 AI가 전담

#### Phase 3: 데이터 큐레이션 (AI = 콘텐츠 에디터)
- 7개 컬러 팔레트 이름/무드/활용처 AI 제안 → 사람이 선택
- 12개 디자이너 격언 AI가 수집 + 번역
- 6개 폰트 페어링 AI 추천 → 디자이너 감수

#### Phase 4: 구현 (AI = 주도 개발자)
- CLAUDE.md 규칙에 따라 AI가 컴포넌트 작성
- 토큰 미준수 코드는 AI 스스로 수정
- 사람은 비주얼 QA와 UX 피드백 전담

### AI-Native의 핵심 가치

> **"AI는 반복과 규칙을 처리하고, 사람은 판단과 감성을 담당한다."**

- AI가 잘하는 것: 일관성, 문서화, 코드 생성, 데이터 큐레이션
- 사람이 잘하는 것: 컨셉 결정, 감성적 판단, 사용자 공감
